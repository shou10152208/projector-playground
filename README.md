# からだで あそぼ！ 🎈

プロジェクターで壁に映して、Webカメラの前で体を動かして遊ぶMRミニゲーム集です。
5歳の子どもと大人が一緒に楽しめるように、「負けない・全部たのしい」設計にしています。

ビルド不要。HTML + JavaScript (Canvas / getUserMedia / Web Audio) で動き、
人体認識用の MediaPipe は `vendor/` に同梱しているので**完全オフライン**で遊べます。

### からだにんしき（人体認識）

カメラ起動後、バックグラウンドで MediaPipe の姿勢認識を起動します。認識できると:

- **手のカーソル**が画面に光って表示され、メニューは「絵に手をかざす」だけで選べます
- しゃぼんだま等の当たり判定が**人体の近くの動きだけ**に反応し、影や照明の変化・通りすがりの映り込みで誤動作しなくなります

認識が使えない環境では、従来どおり「動き検出」だけで自動的に動きます（設定 ⚙️ からON/OFFも可能）。
メニューはマウスクリックやキー `1`〜`7` でも選べます。

## 必要なもの

- プロジェクター(または大きめのテレビ/モニター)
- Webカメラ(ノートPC内蔵カメラでも可)
- モダンブラウザ(Chrome / Edge 推奨)

## 起動方法

カメラを使うため、`file://` 直開きではなく **ローカルサーバー経由** で開いてください。
付属の `serve.py` を使うと、PC用の HTTP と スマホ用の HTTPS を同時に立ち上げます。

```bash
python3 serve.py
#   このPC用 : http://localhost:8000
#   スマホ用 : https://<PCのIP>:8443   (同じWi-Fi)
```

起動時に、アクセス用の URL が一覧表示されます。ブラウザで開き、「はじめる」を押してカメラを許可します。

> 標準ライブラリだけで動くので追加インストールは不要です。HTTPS 用の自己署名証明書は初回に
> `certs/` へ自動生成されます(`openssl` が必要)。

## iPhone / スマホから遊ぶ

iOS Safari は **`https://` か `localhost` 以外ではカメラを使えません**。`serve.py` が HTTPS(ポート 8443)を
提供するので、同じ Wi-Fi のスマホから `https://<PCのIP>:8443` を開きます。
自己署名証明書のため初回は「安全ではない」警告が出ます →『詳細を表示』→『このまま開く』で進めます。

**WSL を使っている場合**、WSL は NAT の内側にいるため、そのままではスマホから WSL のサーバーに届きません。
次のどちらかで到達できるようにします。

- かんたん(推奨): Windows の PowerShell を**管理者として実行**し、付属スクリプトを動かす
  ```powershell
  powershell -ExecutionPolicy Bypass -File setup-iphone-access.ps1
  # 解除: 同じコマンドに -Remove を付ける
  ```
  ポート転送とファイアウォール許可を設定し、スマホで開く URL を表示します。
- または `C:\Users\<名前>\.wslconfig` に `[wsl2]` `networkingMode=mirrored` を書いて WSL を再起動
  (`wsl --shutdown`)すると、WSL のサーバーが PC の LAN IP で直接見えるようになります。

Windows で直接 `python serve.py` を動かせる場合は、上記の設定なしで `https://<PCのLAN IP>:8443` に届きます。

## インターネットに公開する(GitHub Pages)

このアプリは**ビルド不要の静的サイト**なので、GitHub Pages にそのまま置けます。
GitHub Pages は**本物の証明書で HTTPS 配信**されるため、**iPhone のカメラ問題が
証明書警告も WSL のポート転送もなしで解決**します(URL を開くだけ・別ネットワークからもOK)。

> カメラ映像はブラウザ内でのみ処理され、どこにもアップロードされません(`getUserMedia` は端末内処理)。

**ブランチ配信**を使います。`main` ブランチに push するたびに自動で再デプロイされます。

1. リポジトリを GitHub に push する。
2. **Settings → Pages → Build and deployment → Source** を **「Deploy from a branch」**、
   Branch を **`main` / `(root)`** に設定する(`.nojekyll` があるので Jekyll は無効化され、ファイルがそのまま配信されます)。
3. 数十秒後、`https://shou10152208.github.io/projector-playground/` で公開されます。

スマホでもこの URL を開くだけでカメラが使えます(初回にカメラの許可を求められます)。

> 無料プランで Pages を使うにはリポジトリを **Public** にする必要があります
> (Private で使う場合は GitHub Pro)。

## セットアップのコツ

1. **プロジェクターで壁に投影**し、PCの画面をミラーリング(またはブラウザを投影側に移動)して `F` キーで全画面にします。
2. **Webカメラは遊ぶ人に向けます**。投影された壁そのものに向けると、ゲーム画面の動き自体に反応してしまいます。プロジェクターの横あたりから、子どもの全身が映る角度がおすすめです。
3. 反応が鈍い/過敏なときは、右上の ⚙️ →「うごきの かんど」で調整してください。
4. ⚙️ →「カメラの うつりを みる」をONにすると、カメラ映像と検出された動き(緑色)が右下に表示され、カメラの向き調整に便利です。
5. 部屋は真っ暗より、少し明かりがある方がカメラが動きを拾いやすいです。

## しくみ

カメラ映像を 64×36 の粗いグリッドに縮小し、前フレームとの差分(フレーム差分法)を「動きエネルギー」として保持しています。機械学習モデルを使わないので、低スペックPCでも軽快に動き、ネットワーク接続も不要です。映像は左右反転(鏡映し)してから処理するため、プレイヤーの動きと画面上の反応位置が自然に一致します。

効果音はすべて Web Audio API でリアルタイム合成しており、音声ファイルはありません。

## 開発・自動テスト

ヘッドレスブラウザでロジックと描画を自動検証できます(WSL から Windows の Chrome/Edge を使用)。

```bash
bash test/run-tests.sh
```

- **セルフテスト**(`test/selftest.html` + `selftest.js`): 動き検出と各ゲームを合成入力で多数フレーム回し、
  例外がないか・動き検出が正しく立つ/減衰するかを検証します。
- **統合テスト**: `?fakecam=1` という開発用フラグで合成カメラ映像を流し、実際にアプリを起動して
  メニュー・各ゲーム・スマホ縦画面のスクリーンショット(`test/shot-*.png`)を撮ります。

開発用 URL パラメータ(本番動作には影響しません):

| パラメータ | 内容 |
| --- | --- |
| `?autostart=1` | スタート画面を自動で通過 |
| `?fakecam=1` | カメラの代わりに合成映像を入力(カメラなしで動作確認用) |
| `#bubbles` / `#stars` / `#flowers` | そのゲームを直接開く(`autostart` と併用) |

## トラブルシューティング

- **カメラ許可のダイアログが出ない / エラーになる** — `http://localhost:8000` または `https://…:8443` で開いているか確認してください(`file://` や `http://<IPアドレス>` では カメラAPIがブロックされます)。スマホは必ず `https://` で開きます。
- **何もしていないのに反応する** — カメラに投影面やテレビが映り込んでいないか確認し、「うごきの かんど」を下げてください。
- **外付けカメラが使われない** — ⚙️ の「カメラ」から切り替えられます。
- **動きがカクカクする** — 他のタブやアプリを閉じる、ブラウザのハードウェアアクセラレーションを有効にする、を試してください。
